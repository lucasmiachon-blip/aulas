# Git Workflow (simples e seguro)

## Regra
- `main` = versão apresentável (sempre abre)
- Mudanças grandes: branch curta e merge rápido
- Tag por marcos: `v1.0-mvp`, `v1.1-block-calculators`, etc.

## Evitar caos
- Proibido criar 20 branches para micro-ajustes.
- Proibido commit sem mensagem clara.
- Proibido "refatorar tudo" sem necessidade.
